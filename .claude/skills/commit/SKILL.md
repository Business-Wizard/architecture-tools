Inspect the current working copy changes, generate a conventional commit message, and commit using `jj describe`.

## Workflow

1. **Inspect changes:**
   - Run `jj status` to see all modified files
   - Run `jj diff` to read the actual diff

2. **Analyze and generate message:**
   - Identify the primary change type from the allowed list
   - Determine the scope from the files changed
   - Write a concise subject line

3. **Commit:**
   - Run `jj describe -m "<message>"` to set the commit description
   - Run `jj new` to advance to a new working copy
   - Show result with `jj log -r @-`

## Conventional Commits Format

```
<type>(<scope>): <description>
```

**Allowed types** (all lowercase):
- `feat` — new feature or behavior
- `fix` — bug fix
- `refactor` — restructure without behavior change
- `test` — add or fix tests
- `docs` — documentation only
- `chore` — maintenance, deps, tooling
- `build` — build system, Cargo, CI
- `fixup` — minor correction to a recent commit; signaling that it should be squashed and never pushed
- `drop` — **ONLY** for a deliberately temporary commit to expose a bug in CI/production, intended for immediate reversion. NEVER use `drop` for removing code, removing a dependency, deleting a feature, or any permanent deletion.

**Scopes for this project:**
`awt`, `mutations`, `runner`, `graph`, `report`, `cli`, `config`

**Subject line rules:**
- Imperative mood: "add" not "added" or "adds"
- No trailing period
- Max 50 characters (72 hard limit)
- Lowercase after the colon

## Hard Rules

- NEVER use `git` — only `jj` commands
- NEVER add `Co-Authored-By` lines
- NEVER add `Generated by` or AI attribution
- NEVER use `drop` for removing code or features — use `refactor`, `chore`, or `fix` instead
- NEVER amend or rewrite existing commits

## Examples

```
feat(cli): add --dry-run flag to awt run
fix(runner): handle missing verifier binary gracefully
refactor(graph): simplify SCC clustering to use petgraph built-ins
chore(deps): update petgraph to 0.8
test(mutations): cover add_parameter skip constraints
build(cargo): enable thin-LTO in release profile
```
